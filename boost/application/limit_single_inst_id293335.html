Class limit_single_instance_default_behaviour

boost::application::limit_single_instance_default_behaviour — This aspect class ensure that only a single instance of application (process) is running on a given operating system.

Description

Note that the automatic behavior of limit single instance is just evaluated in automatically way when the application is started by the launcher.

Note that user must call lock() to know if a istance of application (process) is running, and only after that the call of is_another_instance_running will return correct value. Call to is_another_instance_running, before call of lock, cause is_another_instance_running to return always false.
